Lakeside Pottery

This redesign was an educational exercise for a visual design class that I took. As such, there was no client contact and the design was not implemented in code. But the project had huge information architecture challenges that I am naturally drawn to, and so was enormously fun to work on. I love to present information in a way that is both deep and skimmable.

Step #1: Analysis

This site is a firehose of visual and text information about pottery and pottery services. But the site's structure was mentally taxing. So the goal of directing users to the course enrollment page would have to be accomplished with a design that produced a logical flow across pages, images, copy, products, and services. Most of the analysis phase was getting enough of a handle on the existing content to put together a simplified site map. I reduced the site from 20+ first-level pages down to just 9. At the same time I set aside the most representative images that I could find for expressing the major sections of copy, and I culled the rest.

  • collected content from existing site, identifying key images
  • drafted a simplified site map: Omnigraffle
  • final report

Step #2: Design

The simplified navigation from the new site map made it easy to know which content needed to be merged onto which new page. And with a ton of information still required to be on the courses page, I took some time to ideate that page's design with different course metadata positioning schemes. I also sketched wireframes and mocked up 3 versions of the home page. In all of these page designs I was carefully selecting, sizing, and placing images to optimally support the copy, not to compete with it as was happening with the existing design. The end result was an inviting design that showcased the stunning pottery visuals of the existing site, but with the right text in the right places to direct users towards the useful resources.
