SED Talks

On this project, an archive site for tech talk presentations, I was tasked with improving an internal website so that it would work with different screen sizes. I also maintained the back-end database for when the web form submission didn't work as expected.

And below are my design improvements...

  • original site desktop screenshot
  • original site mobile screenshot

"Before"

The original design for this website was thrown together by two interns as their side project. Due to time constraints the website was only useable at full-size desktop browswer sizes.

  • back-end connectivity
  • the front end supported full-size desktop browser size only

  • new site desktop screenshot
  • new site mobile screenshot
  • example presentation screenshot

"After"

I restructured the code, adding HTML div tags, CSS table-cell styling, and CSS percentage widths so that the images would align together into the client's desired 8-image grid regardless of browser size. I reconnected the jQuery mouseover behavors to match the new HTML structure. I also created a mobile design and implemented it with a parallel set of CSS rules keyed by CSS media queries into viewport width. This allowed the image and the text to both continue to be useful even on the smaller screen real estate.

  • image rows break in an organized way
  • images scaled proportionally to screen size
  • mobile styling for readability
